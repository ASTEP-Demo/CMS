Appendix A: Understanding Compare Methods

The following compare methods or algorithms are used in the CATO GDPR Search facility to compare the search text with the matching phrase found.

·       Dice Coefficient is a statistic for comparing the similarity of two samples. It searches the database or files by comparing with the search keyword using this algorithm. For more information, refer to the Sorensen Dice Coefficient Wiki page.

·       Levenshtein Distance is the compare method where the distance between two words is the minimum number of single-character edits (insertions, deletions or substitutions) required to change one word into the other.  For more information, refer to the Levenshtein Distance Wiki page.

For example, if you search for "Jonh" instead of "John" with the Levenshtein Distance compare method, you will get the search results with the entries having "John".

·       Longest common subsequence (LCS) The longest common subsequence (LCS) problem is the problem of finding the longest subsequence common to all sequences in a set of sequences (often just two sequences). For more information, refer to the Longest common subsequence problem Wiki page.

·       Fuzzy Search is for full-text searches that allows you to find search terms despite typos, Optical Character Recognition (OCR) errors or small mistakes in your search term. For more information, refer to the Laserfiche Blog page.

·       Double Metaphone looks for variations and inconsistencies in English spelling and pronunciation to produce a more accurate encoding, which does a better job of matching words and names which sound similar.  For more information, refer to the Double Metaphone Wiki page.

For example, if you search for "Joan" instead of "John" (as both sounds similar) with the Double Metaphone compare method, you will get the search results with the entries having "John".
